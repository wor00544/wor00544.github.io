箭头函数

箭头函数： ( ) => { }

注意：
形参如果只有一个的话，小括号可以省略；
形参如果只有0个或者2个及以上，小括号必须有；
逻辑语句只有一条的话，可以省略大括号。如果这条语句是return语句的话，return也要省略掉
逻辑语句两条及以上的话，不可以省略大括号。
箭头函数不能使用arguments
箭头函数不能被当做构造函数

this指向：
自己执行的时候this指向：this指向自己；
隐式this指向：谁最后调用就指向谁；
显示this指向(call,apple)：显示绑定在谁上就指向谁；
new 时this指向：绑定到生成的新对象上；
箭头函数时this指向：只会绑定到外层(函数或者全局)父级的作用域，前面四种的绑定规则不会生效；

异步编程

同步编程：
问题：不合理的模式，会阻塞代码的执行；
好处：代码非常容易理解；

异步编程：
问题：控制反转，代码没有同步好理解；
好处：和浏览器交互，能调用浏览器许多强大的API；

Promise：
问题：大多数人不能很好的理解pormise，没有很好的解决异步流程；
好处：解决了控制反转问题

Generator + Promise:
问题：需要自己写执行器；
好处：用同步的流程写异步，解决了控制反转问题；

块级作用域

块级作用域就是指 { } 中的东西；

暂时性死区(TDZ)：
只要块级作用域内存在let命令，它所声明的变量就“绑定”(binding)这个区域，不再受外部的影响。在块级作用域内，使用let命令声明变量之前，该变量都是不可用的。这在语法上，称为“暂时性死区”，简称TDZ;
本质就是，只要一进入当前作用域，所要使用的变量就已经存在了，但是不可获取，只有等到声明变量的那一行代码出现，才可以获取和使用该变量；

参数的处理

参数默认值：
惰性求值：在默认值里使用函数，只会当默认赋值触发时才会触发函数；
在默认值中使用其他变量要注意先后顺序，其他变量必需先赋值才能被引用；
默认值介于外部作用域和内部作用域之间，不包括内部作用域，所以会引用外部的变量；

剩余运算符：
把散列的元素变成一个集合；
